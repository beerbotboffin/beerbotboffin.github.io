MotoGP Clock & Calendar

MotoGP Clock

Clock & Calendar for the next race, 4 Feb, 2020

The only sport I'm really into these days is bike racing, and Motogp is the pinacle of the sport. I built a clock / calendar to display the countdown to the next race meeting (and also the local time) for my work desk. It's also a bit of a conversation starter.

Description

The clock alternates between the current time and displaying the next race / session (WUP / FP1/2/3 / Qualifying and the Race itself). When I originally coded the calendar, I hard coded the calendar into memory (having previously scraped it from the motogp.com website using python) but alas, the rona turned the season on its head. So, I changed the code to periodically poll my phone to scrape the site for the next session and get an accurate time to set the RTC (in case it has drifted).
The plan is to turn the hotspot on after a race so that the MCU can scrape the next race (and sessions). The reason for this is that the motogp calendar is still 'evolving' as the year and the rona rolls on. I will change the code for 2021, assuming that the calendar is going to be normal then.

My design uses 2 LED matrix displays daisy chained, a DS3231 RTC for keeping the time (useful for the clock function and also calculating the time until the next session and controlled by a NodeMCU all in a delicious in a custom 3D printed case.

Code / firmware

Coding is Arduino and performs all of the functions except timekeeping (which is handled by the RTC).
The code is available here.
